# CLAUDE.md 공통 섹션 (참조용)

이 문서는 `profiles/lite/CLAUDE.md`와 `profiles/standard/CLAUDE.md`에서 **공통으로 들어가는 섹션**을 한 곳에 모아둔 참조본이다. 일반 사용자는 이 파일을 직접 사용하지 않으며, 템플릿 유지관리자가 두 CLAUDE.md를 일관되게 갱신할 때 참조한다.

## 마스터 세션의 절대 원칙 (양 프로필 공통)

- **`/model` 교체 금지**. 마스터는 한 모델을 유지한다. 모델 다양성은 서브에이전트의 `model` 필드로 표현한다.
- **이 CLAUDE.md를 자주 수정하지 않는다**. 캐시 친화 유지가 비용 절감의 핵심.

## 위임 명세 4요소 템플릿 (양 프로필 공통)

```
의도: [무엇을 왜]
제약: [지켜야 할 것 / 건드리면 안 될 것]
완료 기준: [무엇이 되면 끝인가]
관련 파일: [경로 목록 또는 "explorer로 먼저 확인"]
```

## 비용 함정 주의 (양 프로필 공통)

- `model: inherit` 사용 금지. 새 에이전트 추가 시 `model` 명시 확인.
- 서브에이전트끼리 협업을 기대하지 않는다. 자식 서브에이전트 생성 불가. 협업은 마스터 지휘.

## 호출 방법 (양 프로필 공통)

- **자동 라우팅**: description 키워드 매칭.
- **명시적 호출**: `@<agent-name>` 형식.
- **라우팅 스킬 강제**: "라우팅 스킬을 적용해 결정해라" 한 줄.

## 핸드북 참조 (양 프로필 공통)

이 시스템의 단일 진실의 원천은 `HANDBOOK.md`이다. 운영 중 의문이 들면 핸드북의 해당 절을 인용해 결정.

## 프로필별 차이점 (이 부분은 각 CLAUDE.md에서 다르게 작성)

| 항목 | Lite | Standard |
|---|---|---|
| 에이전트 수 | 3 (explorer, implementer, test-writer) | 8 |
| 마스터 직접 처리 영역 명시 섹션 | **있음** (architect/deep-debugger/pr-reviewer/refactorer/doc-writer 영역) | 없음 (모두 위임) |
| Standard 승격 신호 섹션 | **있음** | 없음 |
| 한국어 키워드 매핑 표 | 3행 + "그 외 → 마스터 직접" | 8행 |
| 비용 함정에서 "모든 PR을 reviewer로" 경고 | "lite는 pr-reviewer 부재로 자동 회피" 형태 | "pr-reviewer는 보안·핵심에만" 형태 |
| 검증 강도 라우팅 (test-strategy-routing) 섹션 | 없음 | **있음** |
