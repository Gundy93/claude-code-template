# CLAUDE.md — Lite Profile

이 프로젝트는 **3-에이전트 경량 서브에이전트 시스템**으로 운영된다. 표준 풀셋업의 부분집합.

## 마스터 세션의 절대 원칙

- **`/model` 교체 금지**. 마스터는 한 모델을 유지한다. 모델 다양성은 서브에이전트의 `model` 필드로 표현한다.
- **마스터는 직접 처리 영역이 표준보다 크다**. lite는 의도적인 축소이며, 영역 밖 작업이 누적되면 standard 승격 신호.
- **이 CLAUDE.md를 자주 수정하지 않는다**. 캐시 친화 유지가 비용 절감의 핵심.

## 3개 서브에이전트 요약

| 에이전트 | 모델 | 쓰기 권한 | 핵심 용도 |
|---|---|---|---|
| explorer | haiku | × | 코드베이스 탐색 (거의 모든 첫 호출) |
| implementer | sonnet | ✓ | 명세 명확한 기능 구현·단순 버그 |
| test-writer | sonnet | ✓ | 단위·통합·E2E 테스트 작성 |

## 마스터 직접 처리 영역 (Lite 전용)

표준 프로필에 있는 다음 작업이 lite에서는 **마스터에서 직접** 처리된다:

| 표준 에이전트 영역 | Lite에서의 처리 |
|---|---|
| architect (아키텍처·ADR·API 설계) | 마스터가 ADR 초안 작성. 큰 결정이면 standard 승격 검토. |
| deep-debugger (race·메모리·heisenbug) | 마스터 직접 시도. 가설 3회 실패 시 standard 승격. |
| pr-reviewer (보안·핵심 PR 리뷰) | 마스터 직접 (보안·핵심이면 standard 승격). |
| refactorer (다중 파일 리팩토링) | 마스터 직접 또는 implementer 반복 호출로 쪼개기. |
| doc-writer (README·주석·문서) | 마스터 직접. |

이 영역에서 작업이 **누적되면 standard 승격 신호**. `docs/profile-selection.md`의 "프로필 변경 신호" 표 참조.

## 라우팅 핵심 규칙

**마스터 직접 처리**:
- 한두 줄 답변, 빠른 사실 확인.
- 위 "마스터 직접 처리 영역"의 작업.
- 위임 대상이 마땅치 않은 모호한 작업.

**위임**:
- 코드 탐색·구조 파악 → explorer
- 명세 명확한 기능 구현·단순 버그 → implementer
- 테스트 작성 → test-writer

세부 라우팅은 `.claude/skills/sub-agent-routing/SKILL.md`를 참조.

## 위임 명세 4요소 (이 템플릿을 항상 사용)

```
의도: [무엇을 왜]
제약: [지켜야 할 것 / 건드리면 안 될 것]
완료 기준: [무엇이 되면 끝인가]
관련 파일: [경로 목록 또는 "explorer로 먼저 확인"]
```

## 비용 함정 주의

- **`model: inherit` 사용 금지**. 새 에이전트를 추가할 때마다 `model` 필드를 명시했는지 확인.
- **모든 PR을 마스터가 깊이 리뷰하지 않는다**. 스타일·린트는 도구 영역. 핵심·보안 영역만 마스터가 본다 (lite는 pr-reviewer 부재).
- **서브에이전트끼리 협업을 기대하지 않는다**. 서브에이전트는 자식 서브에이전트를 만들 수 없다. 협업 흐름은 항상 마스터가 지휘.

## Standard 승격 신호 (요약)

다음 신호가 두 번 이상 누적되면 standard 승격 검토:
- deep-debugger급 버그 (race, 미스터리 heisenbug) 2회 발생
- 같은 모듈을 3회 이상 revisit (아키텍처 리뷰 필요 신호)
- 보안·정합성 영역 변경이 자주 발생
- 협업자가 2명 이상이 됨
- 6개월 이상 유지될 가능성

세부는 `docs/profile-selection.md` 참조.

## 한국어 키워드 → 에이전트 매핑 (3-에이전트)

| 한국어 표현 | 매핑 |
|---|---|
| "어디 있어?", "찾아줘", "탐색", "구조 알려줘" | explorer |
| "구현해줘", "기능 추가", "만들어줘", "고쳐줘" | implementer |
| "테스트 작성", "테스트 추가", "단위 테스트" | test-writer |
| "설계해줘", "ADR", "race", "리뷰해줘", "리팩토링", "문서 정리" | **마스터 직접 처리** (또는 standard 승격 검토) |

## 호출 방법

- **자동 라우팅**: 사용자가 자연어로 요청하면 마스터가 description 키워드를 매칭해 서브에이전트 호출.
- **명시적 호출**: `@explorer`, `@implementer`, `@test-writer` 형식.
- **라우팅 스킬 강제**: "라우팅 스킬을 적용해 결정해라" 한 줄.
